# Como pesquisar

Saber pesquisar bem é uma habilidade técnica poderosa. A internet tem muita informação, mas é preciso escolher o que é confiável.

## Onde procurar
- **Documentação oficial**: sempre comece por ela.
- **MDN Web Docs** para HTML, CSS e JavaScript.
- **GitHub** e blogs técnicos para exemplos reais.
- **Stack Overflow** para erros específicos.

## O que escrever na pesquisa
- Use termos simples e diretos.
- Inclua o nome da linguagem ou a ferramenta.
- Adicione o erro exato quando estiver debugando.

### Exemplo
- errado: `html input label`
- certo: `html label input acessibilidade`
- debug: `react useEffect dependency array missing`

## Como avaliar resultados
- Prefira fontes recentes e bem avaliadas.
- Evite respostas sem explicação ou apenas “copy-paste”.
- Verifique se a solução funciona na sua versão da linguagem ou biblioteca.

## Boas práticas ao pesquisar
- Teste as soluções em um ambiente isolado.
- Se achar duas respostas diferentes, compare as vantagens de cada uma.
- Guarde links úteis nos seus próprios arquivos de notas.
